# Marketing Mix 4P/7P

> **Mục tiêu**: Nắm vững framework triển khai marketing từ chiến lược STP thành hành động cụ thể, bao gồm 4P truyền thống và 7P mở rộng cho ngành dịch vụ.

---

## I. MARKETING MIX LÀ GÌ?

### 1. Định nghĩa

**Marketing Mix** là tập hợp các công cụ marketing mà DN sử dụng để **triển khai chiến lược marketing** nhằm đạt mục tiêu tại thị trường mục tiêu. Thuật ngữ được **Neil Borden** đặt ra năm 1953, sau đó **E. Jerome McCarthy** hệ thống hóa thành **4P** năm 1960.

### 2. Từ 4P đến 7P

| Model | Năm | Tác giả | Phạm vi |
|-------|-----|---------|---------|
| **4P** | 1960 | E. Jerome McCarthy | Sản phẩm hữu hình |
| **7P** | 1981 | Booms & Bitner | Mở rộng cho dịch vụ |
| **4C** | 1990 | Robert Lauterborn | Góc nhìn khách hàng |

```
4P (DN-centric)          4C (KH-centric)
─────────────────        ─────────────────
Product    ←──→     Customer Solution
Price      ←──→     Customer Cost
Place      ←──→     Convenience
Promotion  ←──→     Communication

7P = 4P + People + Process + Physical Evidence
```

## II. 4P TRUYỀN THỐNG

### 1. PRODUCT (Sản phẩm)

#### Định nghĩa
**Product** bao gồm tất cả những gì DN cung cấp cho thị trường để đáp ứng nhu cầu KH – bao gồm sản phẩm hữu hình, dịch vụ, trải nghiệm, sự kiện, con người, nơi chốn, tổ chức, thông tin, ý tưởng.

#### 5 cấp độ sản phẩm (Kotler)

```
┌─────────────────────────────────────────────┐
│  5. SẢN PHẨM TIỀM NĂNG (Potential Product) │
│  Mọi cải tiến trong tương lai               │
│  ┌─────────────────────────────────────┐    │
│  │ 4. SẢN PHẨM MỞ RỘNG (Augmented)   │    │
│  │ Bảo hành, giao hàng, lắp đặt       │    │
│  │ ┌─────────────────────────────┐     │    │
│  │ │ 3. SẢN PHẨM KỲ VỌNG       │     │    │
│  │ │ (Expected Product)          │     │    │
│  │ │ Thuộc tính KH mong đợi     │     │    │
│  │ │ ┌─────────────────────┐    │     │    │
│  │ │ │ 2. SẢN PHẨM CƠ BẢN │    │     │    │
│  │ │ │ (Basic Product)      │    │     │    │
│  │ │ │ ┌───────────────┐   │    │     │    │
│  │ │ │ │ 1. LỢI ÍCH   │   │    │     │    │
│  │ │ │ │    CỐT LÕI    │   │    │     │    │
│  │ │ │ │ (Core Benefit) │   │    │     │    │
│  │ │ │ └───────────────┘   │    │     │    │
│  │ │ └─────────────────────┘    │     │    │
│  │ └─────────────────────────────┘     │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

**Ví dụ: Khách sạn**
| Cấp độ | Ví dụ |
|--------|-------|
| 1. Core Benefit | Nơi nghỉ ngơi qua đêm |
| 2. Basic Product | Giường, phòng tắm, khăn |
| 3. Expected | Sạch sẽ, Wi-Fi, TV, điều hòa |
| 4. Augmented | Spa, hồ bơi, concierge, minibar miễn phí |
| 5. Potential | AI personal assistant, VR tour, robot phục vụ |

#### Quyết định về Product

| Quyết định | Chi tiết |
|-----------|---------|
| **Product Line** | Bao nhiêu dòng sản phẩm? (width) |
| **Product Mix** | Tổng thể danh mục SP (width × length × depth × consistency) |
| **Branding** | Tên, logo, brand architecture |
| **Packaging** | Bao bì, thiết kế |
| **Quality** | Mức chất lượng vs giá |
| **Features** | Tính năng nào? Tính năng nào loại bỏ? |
| **Design** | UX/UI, thẩm mỹ, công năng |

### 2. PRICE (Giá)

#### Định nghĩa
**Price** là số tiền KH phải trả để sở hữu/sử dụng sản phẩm. Giá là **P duy nhất tạo doanh thu** (3P còn lại là chi phí).

#### 3 Phương pháp định giá chính

```
                    ┌─────────────┐
            ┌───────│   GIÁ BÁN   │───────┐
            │       └─────────────┘       │
            │               │              │
     ┌──────▼────┐   ┌─────▼─────┐  ┌────▼──────┐
     │ COST-BASED│   │VALUE-BASED│  │COMPETITION│
     │           │   │           │  │  -BASED   │
     │ Giá = CP  │   │ Giá = Giá │  │ Giá = Giá │
     │ + Margin  │   │ trị KH    │  │ đối thủ   │
     │           │   │ cảm nhận  │  │ ± adjust  │
     └───────────┘   └───────────┘  └───────────┘
```

| Phương pháp | Công thức | Ưu điểm | Nhược điểm |
|-------------|----------|---------|-----------|
| **Cost-based** | Giá = Chi phí + % Markup | Đơn giản, đảm bảo lợi nhuận | Bỏ qua nhu cầu KH, cạnh tranh |
| **Value-based** | Giá = Giá trị KH cảm nhận | Tối ưu doanh thu, phù hợp premium | Khó đo lường "perceived value" |
| **Competition-based** | Giá = Giá đối thủ ± điều chỉnh | Nhanh, dễ áp dụng | Phụ thuộc đối thủ, chiến tranh giá |

*(Chi tiết pricing strategies sẽ được phân tích ở document riêng)*

### 3. PLACE (Phân phối)

#### Định nghĩa
**Place** bao gồm tất cả hoạt động đưa sản phẩm từ nhà sản xuất đến tay KH cuối cùng đúng nơi, đúng lúc, đúng số lượng.

#### Kênh phân phối

```
Manufacturer → Consumer                    (Level 0 - Direct)
Manufacturer → Retailer → Consumer         (Level 1)
Manufacturer → Wholesaler → Retailer → Consumer  (Level 2)
Manufacturer → Agent → Wholesaler → Retailer → Consumer (Level 3)
```

#### Chiến lược phân phối

| Chiến lược | Mô tả | Phù hợp | Ví dụ |
|-----------|-------|---------|-------|
| **Intensive** | Có mặt ở MỌI điểm bán | FMCG, convenience goods | Coca-Cola, Mì Hảo Hảo |
| **Selective** | Chọn lọc một số điểm bán | Shopping goods, brand conscious | Samsung, Nike |
| **Exclusive** | Rất ít điểm bán, kiểm soát chặt | Luxury, specialty | Hermès, Rolex |

#### Place trong Digital 4.0: Omnichannel

```
                    ┌──────────────┐
         ┌──────────│   KHÁCH HÀNG │──────────┐
         │          └──────────────┘          │
         ▼                 ▼                  ▼
   ┌──────────┐    ┌──────────────┐    ┌──────────┐
   │  Cửa hàng│    │  E-commerce  │    │  Social  │
   │  vật lý  │◄──►│  Website/App │◄──►│ Commerce │
   └──────────┘    └──────────────┘    └──────────┘
         ▲                 ▲                  ▲
         │          ┌──────────────┐          │
         └──────────│  TRẢI NGHIỆM│──────────┘
                    │  ĐỒNG NHẤT   │
                    └──────────────┘
```

### 4. PROMOTION (Xúc tiến)

#### Định nghĩa
**Promotion** bao gồm mọi hoạt động truyền thông nhằm **thông báo, thuyết phục, và nhắc nhở** KH về sản phẩm/thương hiệu.

#### Promotion Mix (Communication Mix)

| Công cụ | Mô tả | Ưu điểm | Nhược điểm |
|---------|-------|---------|-----------|
| **Advertising** | Quảng cáo trả phí (TV, digital, print, outdoor) | Reach rộng, kiểm soát thông điệp | Đắt, một chiều, ad fatigue |
| **Sales Promotion** | Khuyến mãi, coupon, flash sale, BOGO | Tạo urgency, tăng doanh số ngắn hạn | Giảm brand value, KH chờ giảm giá |
| **PR** | Quan hệ công chúng, media relations, events | Credible, tăng uy tín | Khó kiểm soát, khó đo ROI |
| **Personal Selling** | Bán hàng trực tiếp, tư vấn 1-1 | Cá nhân hóa, xử lý objection | Đắt cost-per-contact, khó scale |
| **Direct Marketing** | Email, SMS, telemarketing, catalog | Targeted, đo lường được | Spam risk, privacy concern |
| **Digital Marketing** | Social media, SEO, content, influencer | Cost-effective, measurable, viral potential | Cần expertise, thay đổi nhanh |

## III. 7P MỞ RỘNG (CHO DỊCH VỤ)

### Tại sao cần thêm 3P?

Dịch vụ có 4 đặc tính (IHIP) mà sản phẩm hữu hình không có:
- **I**ntangibility: Vô hình, không sờ được
- **H**eterogeneity: Không đồng nhất (mỗi lần khác nhau)
- **I**nseparability: Sản xuất và tiêu dùng đồng thời
- **P**erishability: Không tồn kho được

→ Cần 3P bổ sung để quản lý chất lượng dịch vụ.

### 5. PEOPLE (Con người)

| Yếu tố | Giải thích | Ví dụ |
|---------|-----------|-------|
| **Nhân viên phục vụ** | Thái độ, kỹ năng, ngoại hình | Starbucks barista ghi tên lên cốc |
| **Đào tạo** | Training quality, standards | Ritz-Carlton: 250 giờ đào tạo/năm/NV |
| **Văn hóa DN** | Giá trị nội bộ → trải nghiệm KH | Zappos: "Deliver WOW through service" |
| **Khách hàng** | KH tham gia tạo dịch vụ | IKEA: KH tự lắp ráp nội thất |

### 6. PROCESS (Quy trình)

| Yếu tố | Giải thích | Ví dụ |
|---------|-----------|-------|
| **Flow** | Luồng cung cấp dịch vụ | McDonald's: quy trình chuẩn từ order → nhận đồ (90 giây) |
| **Standardization** | Chuẩn hóa chất lượng | KFC: 11 loại gia vị bí mật, quy trình nấu chuẩn toàn cầu |
| **Customization** | Tùy chỉnh theo KH | Subway: KH chọn bánh, nhân, sauce |
| **Technology** | Công nghệ hỗ trợ | Grab: matching algorithm, dynamic pricing |
| **Waiting time** | Quản lý thời gian chờ | Disney: FastPass, ước lượng thời gian chờ |

### 7. PHYSICAL EVIDENCE (Bằng chứng vật lý)

| Yếu tố | Giải thích | Ví dụ |
|---------|-----------|-------|
| **Facility design** | Thiết kế không gian | Apple Store: minimalist, bàn gỗ, ánh sáng ấm |
| **Signage** | Biển hiệu, logo | McDonald's Golden Arches – nhận diện từ xa |
| **Website/App** | Giao diện digital | Airbnb: UX đẹp, ảnh chất lượng cao |
| **Dress code** | Đồng phục nhân viên | Singapore Airlines: Sarong Kebaya iconic |
| **Testimonials** | Reviews, ratings | 4.5 stars trên Google Maps |
| **Tangible cues** | Vật phẩm hữu hình | Khách sạn: dép đi trong phòng, amenities |

## IV. CÁCH ÁP DỤNG VÀ TRIỂN KHAI

### Quy trình xây dựng Marketing Mix

```
Bước 1: STP đã hoàn thành
   ↓ (Biết target là ai, positioning là gì)
Bước 2: Product – Thiết kế SP/DV phù hợp target
   ↓ (Features, quality, design, packaging, brand)
Bước 3: Price – Định giá phù hợp perceived value
   ↓ (Cost-based? Value-based? Competitive?)
Bước 4: Place – Lựa chọn kênh phân phối
   ↓ (Intensive? Selective? Exclusive? Omnichannel?)
Bước 5: Promotion – Xây dựng communication plan
   ↓ (Channel mix, budget allocation, creative)
Bước 6: People – Tuyển dụng, đào tạo nhân sự phù hợp
   ↓ (Nếu ngành dịch vụ)
Bước 7: Process – Chuẩn hóa quy trình
   ↓
Bước 8: Physical Evidence – Thiết kế touchpoints
   ↓
Bước 9: KIỂM TRA CONSISTENCY – 7P có nhất quán không?
   ↓
Bước 10: Triển khai, đo lường, tối ưu
```

### Ma trận kiểm tra Consistency

| | Product | Price | Place | Promotion | People | Process | Physical |
|---|---|---|---|---|---|---|---|
| **Positioning: Premium** | Chất lượng cao ✅ | Giá cao ✅ | Exclusive ✅ | PR, influencer ✅ | Chuyên nghiệp ✅ | Tùy chỉnh ✅ | Sang trọng ✅ |
| **Positioning: Budget** | Cơ bản ✅ | Giá thấp ✅ | Intensive ✅ | Mass ads, promotion ✅ | Hiệu quả ✅ | Nhanh ✅ | Đơn giản ✅ |
| **SAI – Inconsistent** | Cao cấp ❌ | Giá rẻ ❌ | Cửa hàng bình dân ❌ | Flash sale liên tục ❌ | → Khách hàng BỐI RỐI |

## V. ƯU ĐIỂM VÀ NHƯỢC ĐIỂM

### ✅ Ưu điểm:
1. **Toàn diện**: Bao quát mọi công cụ marketing
2. **Đơn giản, dễ nhớ**: 4P/7P framework rõ ràng
3. **Hành động cụ thể**: Mỗi P đều dẫn đến quyết định cụ thể
4. **Linh hoạt**: Điều chỉnh từng P theo tình hình
5. **Kiểm tra nhất quán**: Đảm bảo tất cả P cùng hướng
6. **Phổ biến**: Ngôn ngữ chung của marketing toàn cầu

### ❌ Nhược điểm:
1. **DN-centric**: Nhìn từ góc DN, không phải KH (→ 4C bổ sung)
2. **Thiếu digital**: Framework 1960, chưa tính social media, data, AI
3. **Tĩnh**: Marketing Mix cần liên tục điều chỉnh, không phải "set and forget"
4. **Thiếu mối liên hệ**: 7P thường được xem riêng lẻ, thiếu synergy
5. **Không ưu tiên**: P nào quan trọng nhất? Framework không trả lời
6. **Thiếu Partnership**: Trong digital era, ecosystem partnerships rất quan trọng (nên có P thứ 8?)

## VI. CASE STUDY: GRAB – 7P MARKETING MIX THỜI ĐẠI 4.0

### Bối cảnh
Grab là super app hàng đầu ĐNÁ, từ ride-hailing mở rộng sang food delivery, payments, grocery, insurance.

### 7P Analysis

| P | Chiến lược | Chi tiết |
|---|---|---|
| **Product** | Super App ecosystem | GrabCar, GrabBike, GrabFood, GrabMart, GrabPay, GrabInsure, GrabRewards. Mỗi tính năng là "sản phẩm con" nhắm vào nhu cầu khác nhau |
| **Price** | Dynamic pricing + Subsidized | Surge pricing giờ cao điểm, giá rẻ hơn taxi truyền thống, Flash Sale GrabFood. GrabPay Later (mua trước trả sau) |
| **Place** | Mobile-first, 700+ thành phố | App là "cửa hàng" duy nhất. Có mặt 8 quốc gia ĐNÁ. Cloud kitchen cho GrabFood. GrabMart: giao từ siêu thị/chợ lân cận |
| **Promotion** | Digital-heavy + referral | Voucher app, push notification, referral program ("Mời bạn nhận 50K"), GrabUnlimited subscription, partnership (Grab x Shopee) |
| **People** | Driver-partners + Customer service | 5M+ driver-partners là "face of Grab". GrabAcademy đào tạo tài xế. AI chatbot + human support 24/7 |
| **Process** | Tech-driven seamless experience | Booking → Matching (30s) → Tracking real-time → Payment (auto GrabPay) → Rating. Toàn bộ in-app, frictionless |
| **Physical Evidence** | Digital touchpoints | App UI/UX, driver uniform (xanh lá), helmet branding, in-app receipt, ETA tracking map, review/rating system |

### Consistency Check

```
Positioning: "Everyday everything app" – Tiện lợi, nhanh, đáng tin cậy

Product:  Super app đa dịch vụ               ✅ Consistent
Price:    Giá cạnh tranh, nhiều promotion      ✅ "Affordable convenience"
Place:    Mobile app, mọi lúc mọi nơi          ✅ "Luôn có mặt"
Promotion: Digital-first, data-driven targeting ✅ Tech-savvy
People:   Driver-partner trained, 24/7 support  ✅ Đáng tin cậy
Process:  30s matching, real-time tracking       ✅ Nhanh
Physical: App UX mượt, branding nhất quán       ✅ Professional

→ 7P NHẤT QUÁN với positioning "everyday everything app"
```

### Kết quả & Bài học Marketing 4.0

**Kết quả**:
- GMV (Gross Merchandise Value): $20.5B (2023)
- 35M+ monthly transacting users
- #1 ride-hailing ĐNÁ, #1 food delivery 4/8 thị trường

**Bài học**:
1. **Product là Platform**: Grab không bán 1 sản phẩm mà xây **hệ sinh thái** – mỗi P phải hoạt động cho toàn bộ ecosystem
2. **People = Asset lớn nhất**: 5M+ driver-partners vừa là workforce vừa là brand ambassador. Đào tạo driver = đầu tư marketing
3. **Process = Marketing**: Trải nghiệm 30 giây matching + tracking real-time vừa là Process vừa là cách tạo customer delight
4. **Physical Evidence digital**: Trong digital era, Physical Evidence = App UI, notification design, email template, in-app map
5. **Price as Growth Tool**: Dynamic pricing + subsidized khuyến mãi là cách "mua thị phần" – trade margin for growth

**Hạn chế**: 
- Subsidy-heavy pricing → lỗ nhiều năm, mới lãi Q4/2023
- People (driver) chất lượng không đồng đều → ảnh hưởng brand
- Too many features trong 1 app → UX có thể bị clutter
